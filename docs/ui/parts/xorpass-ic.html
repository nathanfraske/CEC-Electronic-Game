XORPASS-A

The pass-transistor XOR, five layers down

This is the same XOR function as the standard cell, high when the inputs differ, but built a different way. Instead of pulling the output to a rail through a transistor network, it steers the output to a data wire through transmission gates. The trick is to read XOR as a 2-to-1 multiplexer: Y = B ? NOT A : A. When B is low, pass A through to Y; when B is high, pass NOT A. That needs one inverter for the NOT A data, one inverter for the NOT B that completes the gate controls, and two transmission gates, each an NMOS and a PMOS in parallel so the pass is full swing. Eight transistors against the twelve of the and-or-invert build. The defining feature is what the output is tied to: nothing. Only the two inverters touch VCC and GND. The output node is connected, through whichever transmission gate is open, straight to a data node, so it is passed, not pulled. The first tier is the logic symbol, the same XOR shield. The second is the gate as a flow network, a selector that routes the output chamber to one of two reservoirs. The third opens each gate into its NMOS and PMOS valve pair. The fourth is the device as a schematic, the two inverters and the two transmission gates. The fifth is the silicon, the metal-oxide cross-section where the pass transistors tie to data and the output, never to a rail. The transfer curve shows the signature: with B low, Y follows A as a straight diagonal, the analog input passed through with no restoration.
